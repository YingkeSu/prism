# 论文摘要: Multi-UAV Path Planning and Following Based on Multi-Agent Reinforcement Learning

## 基本信息

- **标题**: Multi-UAV Path Planning and Following Based on Multi-Agent Reinforcement Learning
- **作者**: Xiaoru Zhao, Rennong Yang, Liangsheng Zhong, Zhiwei Hou
- **发表年份**: 2024 (Published 11 January 2024, Drones Journal)
- **类型**: 期刊论文 (MDPI)

## 摘要

本文通过提出一种参数共享的离策略多智能体路径规划和跟踪方法，解决了复杂场景中多智能体协作的挑战。与传统的基于网格的方法不同，该方法使用原始激光扫描数据作为输入。系统采用Soft Actor-Critic（SAC）算法作为规划器，集成基于采样的方法生成的路径作为动态"跟随点"。智能体之间的经验共享用于加速策略收敛。提出的方法通过自定义奖励函数专门解决了智能体在初始阶段或目标附近变得静止或过于谨慎的问题。

## 方法论

### 核心算法

- **算法选择**: 使用**Soft Actor-Critic（SAC）**
  - 基于最大熵强化学习框架的离策略Actor-Critic算法
  - 优点：稳定性和探索能力

### 混合规划

- **采样算法**: 结合**自适应信息树（AIT*）**
  - 生成初始预定义路径
  - 这些路径作为DRL规划器的"跟随点"

### 多智能体框架

- **参数共享架构**:
  - 智能体共享公共经验回放缓冲区
  - 允许智能体从彼此的经验中学习
  - 无需显式通信机制

- **感知机制**:
  - 智能体通过LiDAR将彼此视为环境的一部分
  - 避免了复杂的联合动作/状态空间

### 状态空间

包含以下信息：
1. 激光扫描数据（40束）
2. 无人机速度
3. 相对于目标的位置
4. 当前跟随点

### 奖励函数设计

多组件奖励 ($r_{total} = r_{dis} + r_{follow} + r_{crash} + r_{free} + r_{step} + r_{move}$):

1. **距离奖励** ($r_{dis}$): 激励接近目标
2. **跟随奖励** ($r_{follow}$): 激励跟随预定义路径
3. **碰撞惩罚** ($r_{crash}$): 碰撞时的大幅惩罚
4. **自由奖励** ($r_{free}$): 无碰撞时的正向奖励
5. **步数奖励** ($r_{step}$): 控制每步奖励权重
6. **移动奖励** ($r_{move}$): ⭐ **关键创新** - 激励前进以防止局部最优

## 主要贡献

1. **可扩展的仿真环境**
   - 开发了多无人机仿真环境（使用ROS和Rviz）
   - 参数随智能体数量线性增长而非指数增长
   - 避免联合动作/状态空间

2. **参数共享方法**
   - 引入利用所有智能体交互数据的离策略框架
   - 提升连续动作空间中的学习效率

3. **运动激励型奖励**
   - 设计特定的奖励结构
   - 解决DRL中智能体在初始阶段或目标附近保持静止的常见问题

## 实验结果

### 实验设置

- **环境**: 20m × 20m
- **智能体数**: 3个无人机
- **障碍物密度**: 5%到15%

### 收敛性能

- 所有测试方法（基线SAC、混合、混合-move）在约**200个回合**内完成学习并稳定

### 效率

- **"hybrid-move"方法**（完整提出方法）:
  - 相比基线和简单混合版本，实现了**最小回合时间**

### 成功率

- **提出算法**: 所有3个无人机到达目标的**80%成功率**
- **基线方法**: 约为70%成功率
- **性能提升**: 成功率提升约14%

### 障碍物避免

- 仿真展示了成功的实时障碍物避免
- 使用仅基于LiDAR的感知实现智能体间碰撞预防

## 与开题报告的相关性

### 高度相关的技术点

1. **SAC算法应用**
   - 直接验证了开题报告选择SAC的合理性
   - 在连续动作空间和多智能体环境中的有效性

2. **混合规划方法**
   - DRL + 传统采样算法（AIT*）
   - 与开题报告的"RL + 运动基元"思想相似
   - 跟随点概念可类比于运动基元

3. **奖励函数设计**
   - 多组件奖励函数的详细设计
   - 特别是"移动奖励"的设计，解决停滞问题
   - 可为开题报告的奖励函数提供参考

### 可借鉴的实现细节

1. **参数共享经验回放**
   - 如果开题报告考虑多无人机场景，这是重要的架构选择
   - 避免了状态空间爆炸

2. **原始传感器输入**
   - 直接使用LiDAR扫描数据，而非网格化
   - 支持端到端学习的思路

3. **SAC超参数**
   - 稳定在200回合内收敛
   - 可作为开题报告训练时间估算的参考

### 启示

1. **奖励函数的重要性**
   - 详细的奖励工程对训练成功至关重要
   - 需要平衡多个目标（到达、安全、效率）

2. **混合方法的优势**
   - 结合传统方法和DRL可获得更好的性能
   - 验证了开题报告"高层RL + 底层优化"的合理性

3. **停滞问题的解决**
   - 开题报告需要考虑智能体可能停滞或过于谨慎
   - 可借鉴"移动奖励"的设计思路

### 与单无人机场景的差异

虽然本文关注多智能体，但对于开题报告的单无人机场景：

1. **简化版**: 可去除智能体间交互，专注于单智能体
2. **奖励函数**: 简化为单组件奖励（去除智能体间碰撞惩罚）
3. **状态空间**: 移除其他智能体的信息

---

**总结**: 本文为SAC在无人机路径规划中的应用提供了实证支持。奖励函数设计和混合规划思路可直接应用于开题报告。特别是"移动奖励"的设计，为解决RL训练中的停滞问题提供了重要参考。虽然关注多智能体，但核心思想对单无人机场景同样适用。
