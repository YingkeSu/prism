## 3. 关键实验现状

### 3.1 已观察到的规律
- 中等难度（degradation=0.5）下，各方法成功率接近 100%，区分度有限。
- 困难难度下，无可靠性基线碰撞率显著更高。

### 3.2 困难难度三随机种子（timesteps=1000）
- `ours_dynamic`: collision = **0.100 ± 0.108**，time_limit = **0.900 ± 0.108**，reward = **592.43 ± 71.22**。
- `no_reliability`: collision = **0.233 ± 0.062**，time_limit = **0.767 ± 0.062**，reward = **554.50 ± 71.32**。

解释：当前训练步数较低时，可靠性感知分支在“降低碰撞失效”上有正向趋势，但总体成功率仍受收敛不足影响。

## 4. 关于“成功率为 0”的原因判断

在 `hard + degradation=0.5 + timesteps=1000` 配置下，成功率为 0 的原因是组合效应：
1. 场景难度高（目标更难到达，障碍更密集）；
2. 噪声退化增强了策略学习难度；
3. 1000步训练明显不足，SAC 尚未稳定收敛。

补充验证：提高 `max_steps`（200→400）后成功率仍为0，但终止分布变化明显，说明不仅是回合预算问题，核心仍是训练不足。

## 5. 当前风险

- 统计风险：当前 hard 结果虽有多种子，但训练步数低，方差仍较大。
- 资源风险：100k 级实验若仅用 CPU 将显著拉长周期。
- 论证风险：归一化调制独立贡献尚需更强统计证据。

### 6.2 预计使用的公开数据集（校准/泛化评测）
| 数据集 | 模态 | 计划用途 |
|---|---|---|
| TartanAir | RGB / 深度 / IMU / 位姿 | 跨域视觉退化与IMU扰动评估，验证可靠性分数相关性 |
| EuRoC MAV | 双目 / IMU | 校准IMU一致性指标与时序分支稳定性 |

说明：公开数据集主要用于离线校准与泛化验证；RL 主训练仍在当前仿真环境执行，以保持任务与奖励定义一致。

### 6.3 对比基线规划
| 基线 | 类型 | 说明 | 优先级 |
|---|---|---|---|
| `no_reliability` | 项目内 | 直接拼接，不使用可靠性 | P0 |
| `fixed_equal` | 项目内 | 固定等权融合 | P0 |
| `no_norm_modulation` | 项目内消融 | 去除可靠性调制归一化 | P0 |
| `attn_only` | 外部思路改造 | 保留注意力但去掉显式可靠性输入 | P1 |
| `single_modality` | 下界基线 | LiDAR-only / RGB-only / IMU-only | P1 |
| `GMU` | 经典融合门控 | Gated Multimodal Unit | P1 |
| `MoE-gating` | 经典融合门控 | Mixture-of-Experts 门控融合 | P2（资源允许） |

### 6.4 训练规模与交付物
- 最小主实验矩阵：`4 variants x 2 difficulties x 3 seeds = 24 runs`。  
- 预计算力消耗：单次 100k 训练约 `1.0~1.5` 小时（目标机器），主矩阵约 `24~36` 小时。  
- 交付物：主结果表（均值±方差）、显著性检验结果、失败案例与可解释性图。
